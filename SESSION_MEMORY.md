## 关键约束 / 风险
- Word 的自动目录必须依赖标题样式和字段刷新；仅写入“目录”文本不会生成目录
- 当前脚本如果不补 TOC 域，后续再怎么调整 Markdown 内容，导出的 DOCX 仍然会是纯文本目录
- `win32com` 不可用时，Word COM 路线无法直接执行，需要切换到 Pandoc 或先安装依赖
- 新终端会读取到更新后的用户 PATH；当前已打开的旧终端可能需要重开或刷新环境变量
- 当前文档里的 SVG 图在 `python-docx` 里不能直接嵌入，脚本会保留文字占位，图像需要后续单独处理
- 目前细节目录没有真实页码；若要页码，需要 Word 域刷新或额外分页计算，复杂度更高
- 点线目录的页码位置是通过 tab stop 留出来的占位，不是自动生成的实际页码
