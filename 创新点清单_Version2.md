# AntiFraud AI Assistant 创新点清单

## 🟢 简单

### 1. 用户风险趋势

- **实现方式**：遍历 `GetCaseHistory()`，按时间排序聚合每条记录的风险等级，新增 `GET /api/user/risk-trend` 接口返回趋势数组和趋势方向（上升/下降/平稳）已完成
### 2. 案件时序热力图

- **实现方式**：对 `historical_case_library` 表按 `created_at` 月份 + `scam_type` 做 SQL 聚合查询，新增管理员接口 `GET /api/scam/case-library/stats/heatmap` 返回各月各类型案件数量 已完成

### 3. 防诈知识库问答

- **实现方式**：在 Chat 工具列表中加入 `search_similar_cases` 工具，更新系统提示词，使模型在用户询问诈骗知识时主动调用案件库检索并以知识问答形式回答 已完成

## 🟡 中等

### 4. 诈骗链路还原

- **实现方式**：在 `submit_final_report` 的 JSON schema 中新增 `attack_steps: []string` 字段，更新主智能体提示词要求输出骗局步骤，前端渲染成时间线展示  已完成

### 5. 关键词风险标注

- **实现方式**：在 `submit_final_report` schema 中新增 `risk_tokens: [{text, risk_level}]` 字段，主智能体分析完成后对原文中的高危词汇进行标注并返回，前端高亮渲染  已完成

### 6. 多轮对话反诈引导

- **实现方式**：新增一套引导模式系统提示词，当 Chat 检测到高危关键词时切换为主动追问模式，依次引导用户回答"对方如何联系你"、"是否要求下载 APP"、"是否提到安全账户"等关键问题  已完成

### 7. 用户行为异常检测 + WebSocket 实时警报

- **实现方式**：新增独立的 `GET /api/alert/ws` WebSocket 接口，连接存在时启动 goroutine 每 30 秒轮询一次 `history_cases` 表，触发条件为“最近窗口内出现中/高风险记录”时主动推送风险预警，断开连接后 goroutine 自动退出，不改动任何现有分析接口  已完成  已完成

### 8. 个性化记忆系统：用户历史案件向量化召回

- **实现方式**：在用户历史归档时，同步生成案件摘要的 Embedding 向量并存入向量存储系统；新增 `search_user_history` 工具，使智能体拥有“个性化记忆”，能基于语义召回该用户过往遇到的相似骗局，实现跨时空的案件关联分析与深度预警。配合全局“防诈知识库”，构建起“通用+个性化”的双重防护网。  已完成

## 🔴 较复杂

### 9. 反诈知识图谱

- **V1 实现方式（已完成）**：不改数据库结构，直接只读分析 `historical_case_library`，新增管理员接口 `GET /api/scam/case-library/cases/graph`，输出诈骗类型画像与轻量图谱：
  - `scam_type -> target_group` 高频关联
  - `scam_type -> keyword` 高频关联
  - `scam_type <-> scam_type` 类型相似关系
  - 相似分数由 `向量中心余弦相似度 + 关键词重合度 + 目标人群重合度` 加权得到
- **V1 缓存策略（已完成）**：图谱分析结果按 `focus_type + top_k` 维度缓存到 Redis，默认缓存 `5` 分钟；Redis 不可用时自动回退现算，不影响接口可用性。
- **后续可升级点**：
  - 进程内快照缓存图谱结果，进一步降低管理员多次打开时的重复计算成本
  - 增加更多可解释维度（风险等级分布、法律条文、典型话术）
  - 后续若积累足够阶段数据，再升级为更强的流程/关系图谱

### 10. 亲友守护协同预警（规划中）

- **实现方式（规划中）**：新增低耦合“守护关系”子模块，独立维护 `guardian_relations` 与 `guardian_notifications`；提供 `POST /api/guardians/bind`、`GET /api/guardians`、`DELETE /api/guardians/:id` 等接口管理亲友关系。多模态分析链路不直接操作关系表，而是统一产出风险事件，由守护通知服务按阈值消费事件，并向守护人推送简版风险摘要、建议动作与案件详情链接。
- **前端展示方向**：新增“亲友守护”设置入口，支持绑定申请、关系确认、守护状态查看，以及高风险通知列表/抽屉。
- **工程约束**：
  - 所有关系配置、通知模板、接口文档统一使用 `UTF-8` 编码维护；
  - 通过“风险事件 -> 守护通知服务”解耦主分析流程与守护链路，避免跨层直接调用；
  - 对重复绑定、越权查看、通知失败做就地处理，通知失败时降级为站内未读消息，避免连环报错。

### 11. 新型诈骗发现与变种预警（规划中）

- **实现方式（规划中）**：新增管理员接口 `GET /api/scam/patterns/emerging`，按最近时间窗口对案件的 `scam_type / keywords / target_group / embedding` 做聚类与增长分析，识别“与历史标签不完全一致但近期上升明显”的疑似新型诈骗模式，并返回增长幅度、代表关键词、相似旧类型与样例案件。
- **前端展示方向**：管理员侧新增“新型诈骗发现”卡片或榜单，突出显示最近新增模式、影响人群和变化趋势，便于比赛演示“平台治理”能力。
- **工程约束**：
  - 使用只读聚合服务和独立 DTO 输出，不改动现有分析、聊天、图谱接口的主链路；
  - 聚类结果优先缓存到 Redis，缓存不可用时回退现算或基础统计，保证接口可用；
  - 对低样本、噪声数据、聚类异常做阈值过滤与兜底输出，防止误报和级联故障。
