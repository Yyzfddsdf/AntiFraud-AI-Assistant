02 · Agent Interaction

智能体交互主链路

这一部分展开 README 中真正需要解释的“分析原理”：从任务受理、子智能体并发、多工具闭环，到风险等级判定、审核沉淀、历史归档与家庭通知回调，完整还原一条多模态案件分析链。

必经步骤 可选分支 自动触发 异常出口
01Intake

任务受理与排队

入口负责创建任务、写入 pending 状态，并把多模态分析请求交给异步处理链路。

必经
POST /api/scam/multimodal/analyze
state.CreateTask
写入 pending_tasks
queue.processTask
MarkTaskProcessing
02Sensing

子智能体并发分析

不同模态并行提取结构化 evidence，只做感知和摘要，不直接落库、不直接给终态结论。

必经

ImageAgent

对图片内容做视觉元素、话术截图、转账诱导和仿冒界面识别。

VideoAgent

对视频内容中的流程演示、页面操作和诱导动作进行结构化提取。

AudioAgent

对音频中的诱导话术、催促施压与身份冒充线索做文字化归纳。

03Reasoning

MainAgent 工具闭环

MainAgent 把文本、图片、视频、音频 insights 聚合到同一上下文里，再按阶段使用工具补证、评分、判级和生成报告。

核心链路

证据补全

先把“本案”和“外部上下文”补全，再进入真正的风险计算。

query_user_info search_similar_cases search_user_history

画像微调

若聊天或分析过程需要沉淀用户近期状态，可增量更新画像标签。

update_user_recent_tags

决策与报告

先算当前案件分数，再结合历史阈值和命中情况得到最终等级，最后生成结构化报告。

submit_current_risk_assessment resolve_dynamic_risk_level submit_final_report

判级规则在这里完成闭环

MainAgent 不直接手写 risk_score。它先提交结构化风险因子，再由系统计算 current_score；随后结合 historical_score、knowledge base hit、user history hit 得到 dynamic_threshold 和最终 risk_level。

current_score ± 相似命中修正 → compare dynamic_threshold → output risk_level
04Persist

归档、审核与知识沉淀

最终报告生成后，主链路必须归档到用户历史；典型案件则可进入待审核队列，等待管理员确认后再进入正式知识库。

归档必经

主归档路径

这是每个任务的终态步骤，缺了这一步就不算完整闭环。

write_user_history_case history_cases user_history_vectors state.MarkTaskCompleted

可选审核分支

只有在“典型案例”成立时才需要走这条链，强调质量而不是数量。

upload_historical_case_to_vector_db pending_review_cases admin approve / reject historical_case_library
05Exit

回调与异常出口

成功结束后，高风险历史记录会自动触发家庭守护通知；若执行链任意阶段出错，则直接走失败出口。

自动/异常

自动回调

只有高风险历史案件才会触发 family_system.HandleRiskEvent，用于家庭联防通知。

history_cases 高风险事件 family_system.HandleRiskEvent

异常出口

如果排队、模型调用或工具执行失败，任务不再继续推进，直接进入失败态。

queue / tool / model error state.MarkTaskFailed
