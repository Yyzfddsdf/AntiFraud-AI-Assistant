# 软件级创新清单

本文档用于梳理“手机端实时反诈”方案中的软件侧创新点、后端支撑点，以及两者之间的对应关系。目标不是把智能体放进实时主链路，而是构建一套“端侧实时识别与干预、后端策略与联动支撑”的低成本、低耦合、可降级体系。

## 1. 总体定位

### 1.1 核心原则

1. 硬阻断必须由手机端本地执行，不能依赖后端实时返回。
2. 后端承担控制面、情报面、联动面，不承担毫秒级拦截执行。
3. 主判定链路以规则匹配为主，模型和智能体只做增强判断与解释。
4. 所有模块通过标准事件和标准动作通信，避免跨层直接耦合。

### 1.2 目标拆分

1. 软件侧负责采集、识别、干预、上报。
2. 后端负责注册、下发规则、同步情报、接收事件、联动家属、沉淀反馈。
3. 软件与后端通过统一的 `RiskEvent`、`PolicyBundle`、`ActionDecision` 协议协作。

## 2. 软件侧创新点

### 2.1 端侧实时风险感知引擎

统一采集通知、来电、前台应用、页面文本、链接访问等系统信号，形成实时风险输入层。

### 2.2 多源事件标准化总线

将短信、聊天、电话、浏览器、支付页面等不同来源统一映射为标准 `RiskEvent`，为后续规则匹配和后端上报提供统一边界。

### 2.3 本地低成本规则判定引擎

以关键词、正则、黑白名单、时间窗口、序列规则、风险分累积为核心，实现低延迟、低成本、可解释的实时识别。

### 2.4 诈骗链路识别机制

不只看单条消息，而是识别“聊天诱导 -> 打开链接 -> 下载 App -> 打开支付应用 -> 转账”这类连续行为链。

### 2.5 分级干预与软阻断系统

根据风险等级执行静默提醒、悬浮提醒、全屏预警、倒计时确认、来电拦截、恶意域名阻断等动作。

### 2.6 家庭联防协同机制

当用户进入高风险场景时，自动触发家属或守护人通知，形成“个人识别 + 家庭联动”的双层防护。

### 2.7 本地缓存与离线可用机制

规则包、黑名单、情报库、设备策略在本地缓存，确保网络不稳定时仍可维持基本防护能力。

### 2.8 权限失效与能力降级保护

当通知监听、来电识别、VPN、无障碍等能力不可用时，系统应局部降级而非整体失效。

### 2.9 风险解释与用户可理解提示

每次告警都要给出命中规则、风险原因和建议动作，避免用户只看到“高风险”而不知道为什么。

### 2.10 误报反馈与自我修正机制

支持用户标记误报、确认诈骗、补充说明，为后端规则调优与情报更新提供闭环。

## 3. 后端支撑点

### 3.1 设备注册与能力发现

负责注册设备、记录 App 版本、Android 版本、权限状态、模块能力清单、推送标识等信息。

### 3.2 规则包管理中心

负责下发完整规则包和增量规则包，包括关键词、正则、链路规则、动作策略、阈值和冷却时间。

### 3.3 诈骗情报同步中心

负责同步诈骗号码、钓鱼域名、短链、恶意下载源、高风险 App 列表等外部或内部情报。

### 3.4 风险事件接入与归档

接收手机端批量上报的标准化风险事件，做归档、检索、审计、统计与后续分析。

### 3.5 干预动作回执与审计

记录手机端实际执行的动作，如来电拦截、域名阻断、全屏预警、倒计时确认等，便于追溯与策略优化。

### 3.6 家庭守护通知中心

根据手机端上报的高风险事件，向绑定的家庭成员或守护人发出通知，并维护已读、确认、升级处理状态。

### 3.7 反馈闭环与策略优化

接收误报反馈、确认诈骗反馈、人工复核结论，反向优化规则权重、黑名单和策略阈值。

### 3.8 非实时增强分析服务

只对少量高风险、边界、复杂样本做二次语义分析和解释摘要，不进入实时硬阻断主链路。

### 3.9 设备健康监控

通过心跳和状态上报检测设备是否掉权限、掉服务、规则版本落后，及时提醒修复。

## 4. 软件侧与后端对应关系

| 软件侧创新点 | 后端支撑点 | 核心作用 |
| --- | --- | --- |
| 端侧实时风险感知引擎 | 设备注册与能力发现、设备健康监控 | 知道每台设备当前具备哪些采集能力，以及哪些能力已经失效 |
| 多源事件标准化总线 | 风险事件接入与归档 | 统一事件结构，保证端云之间的协议稳定 |
| 本地低成本规则判定引擎 | 规则包管理中心 | 后端负责下发规则，手机端负责执行规则 |
| 诈骗链路识别机制 | 规则包管理中心、风险事件接入与归档 | 后端定义链路规则，端上识别行为序列 |
| 分级干预与软阻断系统 | 干预动作回执与审计 | 手机端执行动作，后端记录动作结果和效果 |
| 家庭联防协同机制 | 家庭守护通知中心 | 手机端触发风险，后端负责通知守护人和状态流转 |
| 本地缓存与离线可用机制 | 规则包管理中心、诈骗情报同步中心 | 后端持续更新，手机端离线缓存并回退使用 |
| 权限失效与能力降级保护 | 设备健康监控 | 端上做降级，后端感知异常并提示用户修复 |
| 风险解释与用户可理解提示 | 非实时增强分析服务、规则包管理中心 | 端上展示命中原因，复杂场景可由后端补充解释摘要 |
| 误报反馈与自我修正机制 | 反馈闭环与策略优化 | 用户反馈回流后端，用于规则和情报持续优化 |

## 8. 设计约束说明

### 8.1 UTF-8

规则文件、接口文档、日志文本、风险说明统一使用 UTF-8，避免中文规则和风险提示在端云间出现编码问题。

### 8.2 低耦合

软件侧与后端只通过标准化协议交互；采集、判定、干预、同步分层独立，避免一个模块修改牵动全局。

### 8.3 防连环报错

任何单点能力失效都必须局部降级；后端接口异常不能阻塞本地实时识别；本地缓存失效时仍要保留最小规则集。
