# 反诈卫士（Sensible AI）服务外包比赛答辩 PPT 设计方案（获奖导向版）

## 1. 设计目标

本方案定位为 **服务外包比赛答辩 PPT**，不是技术文档、不是开发说明书，也不是论文汇报。

表达目标是让评委快速理解三件事：

1. 这个项目解决的是真实问题，而且问题足够重要。
2. 这个方案对用户、社会和服务场景都有明确价值，值得推广。
3. 这个项目不是空想，已经具备真实落地能力和可信的技术支撑。

因此整套答辩的重心不是“技术讲多细”，而是“价值讲多透、落地讲多实、方案讲多可信”。

核心目标只有三个：

1. 第一眼让评委意识到：这不是普通 AI 演示，而是面向真实生活场景的服务型项目。
2. 中段让评委记住：项目同时解决“识别难、提醒晚、缺协同、难持续”的现实问题。
3. 结尾让评委形成稳定印象：项目有价值、有产品形态、有落地基础、值得认可。

建议主答辩页数控制在 **14 页左右**，结构按照：

`项目背景与痛点 -> 用户需求 -> 解决方案 -> 核心功能 -> 用户/社会/商业价值 -> 技术支撑与可靠性 -> 落地与推广前景`

## 2. 总体视觉方向

### 2.1 视觉关键词

建议整体风格采用：

- **安全感**
- **科技感**
- **预警感**
- **系统级闭环感**

不要做成传统学生答辩那种“白底蓝字 + 大段文字”的作业风，也不要做成纯黑客风。最佳方向是：

> 深色科技底 + 高亮预警色 + 少量实机截图 + 大比例架构图/流程图 + 少量情境型概念图

### 2.2 推荐配色

- 主背景色：`#081826`
- 深辅色：`#102B3F`
- 科技青：`#16C2D5`
- 风险橙：`#FF7A2F`
- 成功绿：`#2EB67D`
- 浅灰白：`#F4F7FB`
- 正文字：`#D8E4F0`
- 次级字：`#94A9BC`

配色原则：

1. 大部分页面以深色为主，体现“安全监测系统”的气质。
2. 风险页、预警页、主动守护页，用橙色和红橙做强调。
3. 架构和治理页用青色、蓝色强调专业度。
4. 不要全程大量使用红色，避免显得廉价和杂乱。

### 2.3 字体建议

- 标题：`HarmonyOS Sans SC Bold` 或 `思源黑体 Heavy`
- 正文：`HarmonyOS Sans SC` 或 `思源黑体 Normal`
- 数字/指标：`Bahnschrift` 或 `DIN Alternate`

### 2.4 版式原则

1. 每页只保留 **1 个主视觉中心**。
2. 每页文字控制在 **3-5 个短句要点**，避免整段文字搬上去。
3. 一页中如果已经用了结构图，就不要再塞 3 张截图。
4. 一页中如果用了 2 张以上截图，右侧文字就必须极简。
5. 答辩 PPT 不是产品说明书，要用“强结论 + 强图像”取代“全解释”。

## 3. 图片使用总策略

整套 PPT 建议采用以下图片组合：

1. **概念型图片 2 张**
   - 用于封面和结尾。
   - 不用软件截图，建议用“手机诈骗情境 / 家庭守护 / 风险预警”类高质量概念图或 AI 生成图。
2. **系统图/架构图 5-7 张**
   - 已搬运到 `competition_docs/比赛PPT/assets/svg/`。
   - 这是本项目最强的可信度来源。
3. **移动端截图 4-5 张**
   - 只挑最能说明“真实产品落地”的画面。
   - 不要一股脑堆满所有界面。
4. **后台/治理端截图 1-2 张**
   - 用来证明项目不只是前台 APP，还有平台治理能力。

结论：

- **需要放图片，而且必须放。**
- 但不是“页页都塞截图”，而是“每页只放最能证明这一页结论的图”。
- 本方案执行时按 `单页最多 2 图、优先 1 主图` 控制，避免排版失控。

## 主线总原则

这一版页序不再按“技术模块”展开，而是按服务外包比赛评委最容易接受的逻辑组织：

1. `为什么这个问题重要`
2. `真实用户需要什么`
3. `我们的方案如何解决`
4. `它到底带来了什么价值`
5. `为什么这不是空想，而是能落地的`
6. `为什么值得推广和认可`

主线建议控制为 **14 页**，技术内容压缩到 **2 页主讲 + 1 页辅助证明**。

## 7. 这套 PPT 最重要的取舍建议

当前方案已经明确采用：

- **不增加主线页数，优先减少单页图片数量**
- 主线保持 `14 页`，更符合服务外包比赛答辩节奏
- 技术页压缩为 `2 页主讲 + 1 页辅助证明`
- 高密度截图页已压缩为 `2 图以内`

### 7.1 必须保留的页

1. 封面
2. 背景痛点
3. 用户需求
4. 解决方案
5. 核心功能
6. 用户价值
7. 商业/社会价值
8. 主动守护与家庭联防
9. 技术支撑与可靠性
10. 落地与推广前景
11. 总结

### 7.2 可以压缩的页

- 商业价值页
- 效果验证页
- 推广前景页

如果比赛时间只有 6-7 分钟，这三页可以合并或压缩。

### 7.3 不建议的做法

1. 不要把所有界面截图都塞进去。
2. 不要把答辩做成技术文档朗读。
3. 不要整套都用浅色背景。
4. 不要大量复制项目文档原句到 PPT。
5. 不要堆砌模型名和底层术语，重点应是价值、可靠性和落地能力。

## 8. 最终执行建议

如果后续正式制作 PPT，建议按以下顺序推进：

1. 先定第 1、2、4、7、10、14 页，因为它们决定整套答辩的价值表达。
2. 再补第 3、5、6、8、9、11、12、13 页。
3. 所有截图统一做手机壳或卡片化处理。
4. 所有 SVG 架构图尽量转高清 PNG 再入 PPT，避免现场兼容问题。
5. 演讲时每页只讲一个结论，不要逐条念点。
