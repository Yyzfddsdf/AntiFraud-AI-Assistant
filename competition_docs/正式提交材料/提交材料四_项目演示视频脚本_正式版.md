# 反诈卫士（Sensible AI）项目演示视频脚本

## 一、拍法结论

本视频采用：

1. 一个完整诈骗场景做主线。
2. Android 自动快速识别先提醒用户。
3. 用户自己进入软件发起深度分析。
4. 系统返回正式风险报告与止损建议。
5. 最后补一小段管理端知识治理。

这个拍法的原因很明确：

1. 它符合当前项目真实链路。
2. 它突出“快速提醒”和“正式深度分析”是两条不同层级的能力。
3. 它不会把系统拍成“自动替用户做所有决定”，更符合您文档一贯表述。

## 二、主场景设定

视频主场景统一采用：

`冒充客服 + 虚假征信 + 语音催促`

原因：

1. 容易同时展示文本、截图、语音三类证据。
2. 容易拍出“先被诱导，再被系统提醒，最后用户自己进入软件做正式分析”的流程。
3. 这个场景更适合解释为什么快路径只做提醒，而深度分析需要用户主动进入软件。

## 三、核心原则

这条视频必须严格遵守以下口径：

1. 快速分析是自动的。
2. 快速分析只负责提醒用户存在风险。
3. 快速分析不会自动触发深度分析。
4. 深度分析由用户自己进入软件后主动发起。
5. 主视频主线只体现用户路径，不拍家庭联动触发链路。

## 四、视频总时长

建议总时长控制在：

`3 分 40 秒 - 4 分 20 秒`

这样更稳：

1. 不超过赛题要求的 `5 分钟`。
2. 主线更集中。
3. 不把镜头浪费在与当前主路径无关的能力上。

## 五、拍摄设备与素材

建议准备：

1. 一台 Android 真机
   - 用于录制诈骗诱导、自动快速提醒、用户进入软件、正式分析
2. 一台电脑
   - 用于录制管理员端页面
3. 一个普通用户账号
   - 用于移动端演示
4. 一个管理员账号
   - 用于管理员端演示
5. 一段诈骗语音
   - 内容包含征信恐吓、立即处理、催促操作
6. 一张诈骗截图
   - 例如伪造客服通知、仿冒征信异常页面
7. 一段诈骗文本
   - 例如短信或聊天消息

## 六、拍摄前准备

正式录制前，先确认以下状态：

1. 后端服务已启动。
2. `frontend/desktop-vue` 已启动并可登录。
3. Android 原生端已安装并可正常登录。
4. Android 端悬浮窗、通知权限、无障碍能力已提前配置完成。
5. 诈骗截图、诈骗文本、诈骗语音已放入手机。
6. 管理端准备好案例审核页或统计页。

## 七、视频结构与分镜脚本

### 7.1 0:00 - 0:15 开场

**画面**

1. 项目标题页或手机主界面。
2. 系统名称上屏。

**旁白**

“这是反诈卫士，一套基于多模态智能体的反诈智能助手。它既能在手机使用过程中自动做快速风险提醒，也支持用户进入软件发起正式深度分析。”

**字幕**

`反诈卫士（Sensible AI）`

`自动提醒 + 用户主动深度分析`

### 7.2 0:15 - 0:40 诈骗诱导出现

**画面**

1. 手机收到伪造客服消息。
2. 打开一张可疑征信异常截图。
3. 来电界面出现，播放诈骗语音。

**旁白**

“这里演示一个典型高发场景。用户先收到冒充客服的信息，随后又看到征信异常类页面，并接到催促处理的诈骗电话。”

**诈骗语音台词**

“您好，您名下账户存在异常贷款记录。如果今天不处理，将影响个人征信，请立即配合我们操作。”

**字幕**

`诈骗诱导开始`

### 7.3 0:40 - 1:05 自动快速分析提醒

**画面**

1. Android 端出现自动快速提醒。
2. 展示悬浮窗或通知提醒。
3. 强调提醒内容是“建议进入软件进一步分析”。

**旁白**

“当系统在真实手机使用路径中检测到明显风险时，会先自动做一次快速分析。但这一步只负责提醒用户，不会直接替用户发起正式深度分析。”

**建议上屏文案**

`检测到高风险特征`

`建议进入反诈卫士进行正式分析`

`请勿立即转账、共享屏幕或提供验证码`

### 7.4 1:05 - 1:35 用户进入软件

**画面**

1. 用户点击提醒。
2. 进入反诈卫士 App。
3. 从主页进入分析入口。

**旁白**

“在收到自动提醒后，用户再自己进入软件，决定是否发起正式分析。这一点很重要：快速提醒和深度分析是分层设计，不是一个动作自动替代另一个动作。”

**字幕**

`用户主动进入软件`

`用户主动发起深度分析`

### 7.5 1:35 - 2:15 多模态深度分析

**画面**

1. 用户上传文本、截图、语音证据。
2. 提交分析任务。
3. 展示分析中、报告生成中的过程。

**旁白**

“正式分析阶段，用户把文本、截图和语音等证据主动提交给系统。随后由多模态子智能体完成取证，再由主智能体做统一编排、评分和风险判断。”

**字幕**

`文本 + 图片 + 音频`

`子智能体取证 -> 主智能体编排 -> 风险评分 -> 报告生成`

### 7.6 2:15 - 2:55 分析结果返回给用户

**画面**

1. 展示结构化结果页。
2. 重点放大诈骗类型、风险等级、关键证据、止损建议。

**旁白**

“分析完成后，系统向用户返回正式结构化结果。这里不仅告诉用户风险等级，还明确说明诈骗类型、关键证据和下一步止损建议。”

**建议上屏字段**

1. `风险等级：高`
2. `诈骗类型：虚假征信类`
3. `关键证据：冒充客服、征信恐吓、催促立即处理`
4. `建议动作：停止转账、停止共享屏幕、联系官方渠道核实`

### 7.7 2:55 - 3:20 用户侧预警与止损

**画面**

1. 展示用户侧高风险提示。
2. 展示报告页中的防范建议。
3. 可以补一个“用户取消操作”的动作。

**旁白**

“项目在这一步的重点，是把风险判断真正转化为用户可执行的止损动作。也就是说，系统不是只做识别，而是要帮助用户在关键时刻停下来。”

**字幕**

`高风险预警`

`立即停止当前操作`

### 7.8 3:20 - 3:45 管理端知识治理

**画面**

1. 切到管理员端。
2. 展示案例列表、审核页或统计页。

**旁白**

“前台识别之外，项目还具备管理员端治理能力。案例审核、知识沉淀和统计分析会继续支撑后续知识扩充和系统优化。”

**字幕**

`管理端：案例审核 + 知识沉淀 + 统计分析`

### 7.9 3:45 - 4:05 技术点补充

**画面**

1. 快速切 Agent 编排图。
2. 再切长短期记忆或动态风险分级图。

**旁白**

“在技术上，系统通过多智能体编排，把图像、音频、视频等模态证据汇总到主智能体；再结合向量检索、用户历史和动态风险分级，提升判断的稳定性和个性化程度。”

**字幕**

`Agent 编排`

`向量检索`

`长短期记忆`

`动态风险分级`

### 7.10 4:05 - 4:20 收尾

**画面**

1. 回到项目总览页或标题页。

**旁白**

“反诈卫士要解决的，不只是一次诈骗识别，而是帮助用户在真实生活中更早发现风险，并通过自动提醒与主动深度分析形成更可靠的反诈闭环。”

**字幕**

`自动提醒`

`主动分析`

`结构化止损建议`

## 八、视频里必须保留的重点字幕

建议固定保留以下字幕：

1. `诈骗诱导开始`
2. `自动快速分析提醒`
3. `提醒用户进入软件进行正式分析`
4. `用户主动进入软件`
5. `用户主动发起深度分析`
6. `高风险结构化结果`
7. `管理端知识治理`

## 十、拍摄注意事项

1. 不把快速提醒拍成自动发起深度分析。
2. 不把用户路径拍成家庭联动路径。
3. 用户进入软件和主动点击分析入口这两个动作必须拍清楚。
4. 风险报告页面必须至少停留 `4-6 秒`，让评委能看清。
5. 管理端只拍一小段，用来证明项目具备治理闭环，不抢主线。

## 十一、最终建议

这条视频最重要的一句话是：

`先自动提醒，再由用户自己进入软件做正式深度分析。`

只要这条主线拍清楚，视频就会和您现有文档保持一致。
