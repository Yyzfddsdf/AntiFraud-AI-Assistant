# 反诈卫士（Sentinel AI）项目简介 PPT 正式版

## 使用说明

- 本文件对应比赛提交材料二“项目简介 PPT”。
- 本版按“**比赛提交版**”而不是“现场答辩讲稿版”组织。
- 目标是让评委在**独立翻阅 PPT** 时也能完整理解项目。
- 建议控制为 **20 页**，满足“不超过 20 页”的要求。
- 每页均包含：
  - 页面定位
  - 推荐版式
  - 图片建议
  - 可直接上屏的正式文案
- 本文件不再以口播为中心，而以“页面自解释”和“材料完整性”为中心。
- 项目代码仓库网址：
  - `https://github.com/Yyzfddsdf/AntiFraud-AI-Assistant.git`

## 整体组织原则

整套 PPT 按比赛要求和阅读逻辑组织为：

`目录 -> 项目基本信息 -> 问题背景 -> 用户需求 -> 目标与解决思路 -> 总体解决方案 -> 核心功能 -> 核心技术路线 -> Agent编排架构 -> 风险判断机制 -> 多端承载 -> 业务模式 -> 人员组织框架 -> 可行性分析 -> 评测结果与指标达成 -> 项目价值与总结`

相较上一版，本版做了以下调整：

1. 去掉强口播感表达，增强提交材料的正式性。
2. “业务模式、人员组织框架、可行性分析”不再压缩在同一页。
3. 技术部分拆为“核心技术路线”“Agent 编排架构图”“关键机制”和“多端承载”四个层次。
4. 每页尽量形成“标题即结论，正文即依据”的自解释结构。
5. 图片以证据型图片为主，概念图为辅。
6. 允许加入短视频和语音信息时，应优先服务“增强可信度”和“增强场景感”，而不是为了炫技增加材料负担。
7. 企业要求中的 `Agent编排架构图`、`核心技术`、`评测指标` 必须在页标题或图注层直接出现，不依赖评委自行概括。

### 分组起始页样式建议

本版不单独增加“章节过渡页”，原因是总页数需严格控制在 `20 页以内`。  
因此只在少数分组起始页上加强标题层级，用版式节奏替代额外页数。

建议把以下页面作为分组起始页处理：

1. `第 3 页：项目基本信息`
2. `第 8 页：总体解决方案`
3. `第 12 页：核心技术路线`
4. `第 16 页：业务模式`

分组起始页建议统一使用以下处理方式：

1. 标题字号比普通页再大一档，形成明显的阅读停顿感。
2. 标题上方可增加一行很小的分组提示字，例如：
   - `项目认知`
   - `方案与产品`
   - `技术实现`
   - `落地与总结`
3. 顶部或左侧加入一条细色带、短分隔线或轻编号点，用于强化阅读节奏，不单独制造新的命名体系。
4. 背景仍然使用正文统一纯背景，不另起新背景体系，避免整套视觉失控。
5. 除分组起始页外，其余正文页保持常规标题层级，不再重复做强装饰。

## 多媒体使用策略

比赛允许在 PPT 中加入短视频及语音信息，这一项不应浪费，但也不能滥用。建议按“**少量、高价值、强证明**”的原则使用。

### 一、图片策略

图片分为三类：

1. **证据型图片**
   - 优先级最高
   - 包括系统架构图、流程图、手机截图、管理员端截图、统计图
   - 用于证明项目真实存在、真实可用、真实可讲

2. **概念型图片**
   - 只在少数页面使用
   - 适合封面、项目背景、总结页
   - 用于建立情境和氛围，不用于证明能力

3. **解释型图片**
   - 例如重新绘制的机制图、判断流程图、动态阈值示意图
   - 用于把较复杂技术逻辑讲清楚

### 二、短视频策略

建议最多放 **2 段短视频**，总时长控制在 **30 秒以内**。短视频的作用不是展示花哨动画，而是快速证明“这件事真的会发生、真的能跑起来”。

推荐的两个视频位如下：

1. **Android 主动守护短视频**
   - 建议时长：`8-12 秒`
   - 适合放在第 10 页
   - 建议内容：
     - 用户浏览可疑界面
     - 悬浮球或后台提醒触发
     - 弹出高风险提示
     - 如有条件，再接 1 秒家庭联防提示
   - 作用：
     - 比单张截图更能证明“主动守护”不是静态界面，而是真正的事中干预能力

2. **管理员端治理短视频**
   - 建议时长：`10-15 秒`
   - 适合放在第 11 页
   - 建议内容：
     - 案例送审列表
     - 管理员审核操作
     - 案例入库或统计页面切换
   - 作用：
     - 证明项目不仅有用户端，也有后台治理闭环

### 三、语音与提示音策略

语音和提示音不能滥放，否则会让提交版显得杂乱。建议只在以下两种情况下使用：

1. **诈骗语音样本片段**
   - 建议时长：`3-5 秒`
   - 适合第 4 页项目背景或第 13 页技术实现方案一的角标说明
   - 用途：
     - 强化“诈骗已是多模态问题”这一判断
   - 注意：
     - 只建议作为可选附件或点击播放按钮，不建议自动播放

2. **风险提示音 / 守护提醒音**
   - 建议时长：`1-2 秒`
   - 适合第 10 页 Android 主动守护页
   - 用途：
     - 提升主动提醒的场景真实感
   - 注意：
     - 音效必须克制，不宜突兀

### 四、容量控制建议

因为提交文件大小限制为 `50M`，多媒体素材应严格控制：

1. 图片优先用压缩后的 PNG 或高质量 JPG。
2. 视频建议导出为 `MP4 H.264`，单段控制在 `5-12M`。
3. 音频建议导出为 `AAC/MP3`，单段控制在 `300KB-1MB`。
4. 总体建议：
   - 图片 + PPT 本体约 `15-20M`
   - 视频总量约 `10-20M`
   - 预留至少 `10M` 冗余空间

### 五、最终建议

最优方案不是“把允许放的视频和语音都塞进去”，而是：

1. **封面和总结页**用概念图提升气质
2. **核心功能、Android 端、管理员端**用真实截图和结构图建立可信度
3. **第 10 页和第 11 页**各放 1 段短视频，把项目的动态能力亮出来
4. 语音只作为可选增强，不作为主线依赖

这样既能充分利用比赛允许的多媒体优势，又不会让 PPT 失去正式材料应有的稳定感
