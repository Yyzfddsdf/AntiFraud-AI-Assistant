# 反诈卫士（Sensible AI）项目使用指南

## 一、文档目的

本文档用于配合比赛提交包说明项目的启动方式、账号准备、核心功能使用路径和推荐演示顺序，确保评委或复核人员可以快速完成部署与演示。

## 二、运行环境

建议运行环境如下：

1. Windows 10/11 或可运行 Go 的等效环境。
2. Go 1.25。
3. Redis 6.x 及以上。
4. 可访问外部模型服务或兼容 OpenAI 协议的模型网关。
5. 本地可写目录，用于 SQLite 数据库文件与缓存文件。
6. 运行 Android 原生端时，建议准备 Android Studio 与 Android 8.0 及以上真机或模拟器。

## 三、关键配置

部署时主要关注以下内容：

1. 启动入口：`cmd/api/main.go`
2. 主配置文件：`internal/platform/config/config.json`
3. 主业务数据库：`DB/auth_system.db`
4. 历史案件库：`DB/historical_case_library.db`

建议优先检查以下环境变量：

1. `PORT`
2. `JWT_SECRET`
3. `INVITE_CODE_ADMIN`
4. `DB_PATH`
5. `HISTORICAL_CASE_DB_PATH`

## 四、启动步骤

当前仓库采用前后端分离结构，运行时至少包含后端服务；前端与 Android 原生端按需单独启动。

前端/端侧工程当前分为三类：

1. `frontend/desktop-vue`：独立桌面端 Vue 工程，承载管理员端。
2. `frontend/mobile-vue`：独立移动端 Vue 工程，承载移动端 Web 页面。
3. `Android`：原生 Android 工程，承载悬浮窗快捷识别、后台提醒、无障碍自动守护等主动防护能力。

移动端部署方式分为两条路径：

1. Web 移动端通过 `frontend/mobile-vue` 单独启动。
2. Android 原生端可自行 build，也可直接安装我们提供的 APK。

### 4.1 安装依赖

在项目根目录执行：

```bash
go mod tidy
```

### 4.2 确认 Redis 可用

请确认 `internal/platform/config/config.json` 中的 `redis.addr` 可达，且 Redis 服务已启动。

### 4.3 启动服务

在项目根目录执行：

```bash
go run ./cmd/api
```

默认访问地址如下：

1. API 基地址：`http://localhost:8081/api`
2. 后端服务基地址：`http://localhost:8081`

### 4.4 启动桌面端前端

管理员端前端在 `frontend/desktop-vue` 目录执行：

```bash
npm install
npm run dev
```

默认开发地址为：`http://localhost:5173`

### 4.5 启动移动端 Web 前端

Vue 版移动端前端在 `frontend/mobile-vue` 目录执行：

```bash
npm install
npm run dev
```

默认开发地址为：`http://localhost:5174`

说明：

1. `desktop-vue` 与 `mobile-vue` 均为独立前端工程，需要单独启动。
2. 两个前端工程通过 Vite 代理访问后端 `/api`，因此后端服务需要先启动。

### 4.6 使用 Android 原生端

Android 原生端代码位于 `Android/` 目录，当前仓库内已包含完整 App 模块代码，例如：

1. `Android/app/build.gradle.kts`
2. `Android/app/src/main/AndroidManifest.xml`
3. `Android/app/src/main/java/com/example/myapplication/MainActivity.kt`
4. `Android/app/src/main/java/com/example/myapplication/QuickAnalyzeOverlayService.kt`
5. `Android/app/src/main/java/com/example/myapplication/AccessibilityAutoAnalyzeService.kt`

Android 端有两种使用方式：

1. 自行构建：
   - 使用 Android Studio 打开 `Android/` 目录
   - 根据需要调整 `Android/app/src/main/assets/config/app_config.json`
   - 连接真机或启动模拟器后进行编译、安装和运行
2. 直接安装：
   - 直接安装我们提供的 APK 进行演示
   - 这种方式更适合比赛演示或快速复核，不需要现场重新编译

说明：

1. Android 原生端主要承接悬浮窗快捷识别、后台提醒、无障碍自动守护等主动防护能力。
2. 基础联调可只启动后端与 Web 前端；完整展示“事中守护”时使用 Android 原生端。

## 五、账号准备

建议准备三类账号：

1. 普通用户：用于多模态分析、风险历史、聊天辅助、主动提醒演示。
2. 守护人用户：用于家庭联动通知演示。
3. 管理员用户：用于案例审核、知识库运营和后台统计演示。

家庭联防演示建议至少准备 2 个普通用户并提前建立守护关系。

## 六、核心使用路径

### 6.1 快速识别

调用 `POST /api/scam/image/quick-analyze`，输入单张截图，系统同步返回 `risk_level` 与 `reason`。

### 6.2 多模态深度分析

调用 `POST /api/scam/multimodal/analyze`，输入文本并可附带图片、音频或视频内容。系统异步完成：

1. 子智能体多模态取证
2. 主智能体工具编排
3. 当前案件评分
4. 用户历史与画像读取
5. 动态风险分级
6. 最终结构化报告
7. 历史归档与高风险联动

### 6.3 风险历史与总览

普通用户可通过：

1. `GET /api/scam/multimodal/history`
2. `GET /api/scam/multimodal/history/overview`

查看历史案件、风险等级分布与风险趋势。

### 6.4 家庭联防

高风险案件产生后，守护人可通过家庭通知 WebSocket 接收即时提醒。

### 6.5 管理端治理

管理员可通过以下接口完成案例治理：

1. `GET /api/scam/review/cases`
2. `GET /api/scam/review/cases/:recordId`
3. `POST /api/scam/review/cases/:recordId/approve`

并可继续查看案件库总览、图谱与区域风险态势。

## 七、启动后核对项

服务启动完成后，建议按以下顺序核对关键能力是否可用：

1. 后端接口可访问，`/api` 请求返回正常。
2. `frontend/desktop-vue` 可正常登录并进入管理员端页面。
3. `frontend/mobile-vue` 可正常登录并进入移动端页面。
4. 单图快速识别接口可返回 `risk_level` 与 `reason`。
5. 多模态深度分析任务可正常创建、完成并生成结构化结果。
6. 风险历史与总览接口可正常返回历史记录与趋势数据。
7. 家庭联防通知链路可正常触发。
8. Android 原生端的悬浮窗、后台提醒或无障碍自动守护能力可按预期运行。

## 八、补充说明

本项目的赛题核心亮点不在附加聊天能力，而在深度分析主链路中对“当前案件上下文 + 用户历史记忆 + 用户画像”的联合建模。评委如需核对赛题中“长短期记忆机制”的实现，请优先查看：

1. 主智能体工具闭环
2. 动态风险分级逻辑
3. 测评集与评估报告

## 九、结论

当前项目已具备完整后端服务、多模态深度分析、主动预警、家庭联防和知识治理能力。本文档的目标是确保比赛提交包中的可运行部分能够被快速部署和稳定演示。
