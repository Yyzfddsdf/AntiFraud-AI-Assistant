# 反诈卫士（Sentinel AI）项目使用指南

## 开源仓库地址

项目代码仓库网址：`https://github.com/Yyzfddsdf/AntiFraud-AI-Assistant.git`

## 一、文档目的

本文档用于配合比赛提交包说明项目的启动方式、账号准备、核心功能使用路径和推荐演示顺序，确保评委或复核人员可以快速完成部署与演示。

## 二、运行环境

建议运行环境如下：

1. Windows 10/11 或可运行 Go 的等效环境。
2. Go 1.25。
3. Redis 6.x 及以上。
4. 可访问外部模型服务或兼容 OpenAI 协议的模型网关。
5. 本地可写目录，用于 SQLite 数据库文件与缓存文件。
6. FFmpeg 用于视频和音频处理（多模态分析必需）。
7. 运行 Android 原生端时，建议准备 Android Studio 与 Android 8.0 及以上真机或模拟器。

## 三、关键配置

部署时主要关注以下内容：

1. 启动入口：`cmd/api/main.go`
2. 主配置文件：`internal/platform/config/config.json`
3. 安全配置：`internal/modules/login/domain/settings/security.go`
4. 主业务数据库：`DB/auth_system.db`
5. 历史案件库：`DB/historical_case_library.db`
6. Nginx 配置：`nginx/antifraud-ip.conf`

### 配置文件加载逻辑说明

#### 1. `internal/platform/config/config.json`（主配置文件）

该文件包含系统运行所需的各类配置，加载逻辑如下：

**配置优先级（从高到低）：**
1. 环境变量（最高优先级，覆盖配置文件）
2. `config.json` 中的配置值
3. 代码默认值（最低优先级）

**关键配置项及加载规则：**

| 配置项 | 配置路径 | 加载逻辑 |
|--------|----------|----------|
| 各智能体 API Key | `agents.*.api_key` | 环境变量 `AGENT_*_API_KEY` > config.json > 报错 |
| Embedding API Key | `embedding.api_key` | 环境变量 `EMBEDDING_API_KEY` > config.json > 报错 |
| 聊天 API Key | `chat.api_key` | 环境变量 `CHAT_API_KEY` > config.json > 继承主智能体配置 |
| Tavily API Key | `tavily.api_key` | 环境变量 `TAVILY_API_KEY` > config.json > 报错 |
| **FFmpeg 路径** | `media_tools.ffmpeg_path` | 环境变量 `FFMPEG_PATH` > config.json > **默认 `"ffmpeg"`（使用系统 PATH）** |
| **FFprobe 路径** | `media_tools.ffprobe_path` | 环境变量 `FFPROBE_PATH` > config.json > **默认 `"ffprobe"`（使用系统 PATH）** |
| Redis 地址 | `redis.addr` | config.json > 默认 `127.0.0.1:6379` |

**FFmpeg 配置说明：**
- 如果 `config.json` 中 `media_tools.ffmpeg_path` 配置为具体路径（如 `/usr/bin/ffmpeg`），则使用该路径
- 如果配置为空字符串或省略，则自动使用默认值 `"ffmpeg"`，此时会从系统 PATH 中查找
- 环境变量 `FFMPEG_PATH` 和 `FFPROBE_PATH` 可覆盖配置文件中的设置

#### 2. `internal/modules/login/domain/settings/security.go`（安全配置）

该文件定义安全相关常量，加载逻辑如下：

| 配置项 | 环境变量 | 默认值 | 说明 |
|--------|----------|--------|------|
| JWT 签名密钥 | `JWT_SECRET` | `change_me_to_a_strong_secret_in_production` | **生产环境必须通过环境变量设置** |
| 管理员邀请码 | `INVITE_CODE_ADMIN` | `Secret_Admin_Invite_Code_2026` | **生产环境建议通过环境变量设置** |

**加载规则：** 优先读取环境变量，若未设置则使用代码中的默认值。

**其他安全常量（代码硬编码，不可通过环境变量配置）：**

| 常量名称 | 默认值 | 说明 |
|----------|--------|------|
| `JWTExpireDuration` | 24 小时 | JWT 令牌默认有效期 |
| `CaptchaCodeLength` | 5 | 验证码字符长度 |
| `CaptchaTTL` | 3 分钟 | 验证码有效期 |
| `RateLimitWindow` | 1 秒 | 限流统计时间窗口 |
| `RateLimitMaxRequests` | 100 | 单个 IP 窗口期最大请求数 |
| `ActiveTokenLimit` | 2 | 单用户允许同时保留的最近活跃 token 数量上限 |

**说明：** 以上常量为代码中硬编码的默认值，如需修改必须直接编辑 `security.go` 文件后重新编译。

### 建议优先检查的环境变量

1. `PORT`：服务端口，默认 `8081`
2. `JWT_SECRET`：JWT 密钥（生产环境必须设置）
3. `INVITE_CODE_ADMIN`：管理员升级邀请码（生产环境建议设置）
4. `DB_PATH`：主业务库路径（默认 `DB/auth_system.db`）
5. `HISTORICAL_CASE_DB_PATH`：历史案件库路径（默认 `DB/historical_case_library.db`）
6. `FFMPEG_PATH` / `FFPROBE_PATH`：可选，覆盖配置文件中的 FFmpeg 路径

## 四、启动步骤

当前仓库采用前后端分离结构，运行时至少包含后端服务；前端与 Android 原生端按需单独启动。

前端/端侧工程当前分为三类：

1. `frontend/desktop-vue`：独立桌面端 Vue 工程，承载管理员端。
2. `frontend/mobile-vue`：独立移动端 Vue 工程，承载移动端 Web 页面。
3. `Android`：原生 Android 工程，承载悬浮窗快捷识别、后台提醒、无障碍自动守护等主动防护能力。

移动端部署方式分为两条路径：

1. Web 移动端通过 `frontend/mobile-vue` 单独启动。
2. Android 原生端可自行 build，也可直接安装我们提供的 APK。

### 4.1 获取代码

克隆仓库到本地：

```bash
git clone https://github.com/Yyzfddsdf/AntiFraud-AI-Assistant.git
cd AntiFraud-AI-Assistant
```

### 4.2 安装依赖

在项目根目录执行：

```bash
go mod tidy
```

### 4.2 确认 FFmpeg 已安装

多模态分析功能依赖 FFmpeg 进行视频和音频处理。请确保 FFmpeg 已安装并添加到系统 PATH：

```bash
ffmpeg -version
```

### 4.4 确认 Redis 可用

请确认 `internal/platform/config/config.json` 中的 `redis.addr` 可达，且 Redis 服务已启动。

### 4.4 配置 Nginx（生产部署）

生产环境建议使用 Nginx 统一代理前后端服务：

1. 将前端构建产物放置到 `nginx/build/` 目录（desktop 和 mobile 子目录）
2. 复制 `nginx/antifraud-ip.conf` 到 Nginx 配置目录
3. 根据实际部署路径修改配置文件中的 `root` 和 `alias` 路径
4. 重启 Nginx 服务

Nginx 配置实现了：
- 同一端口（80）提供前后端服务
- `/api/` 路径代理到后端服务（默认 8081 端口）
- 根路径 `/` 根据 User-Agent 自动判断返回桌面端或移动端页面
- `/m/` 路径提供移动端 Web 页面

### 4.6 启动服务

在项目根目录执行：

```bash
go run ./cmd/api
```

默认访问地址如下：

1. API 基地址：`http://localhost:8081/api`
2. 后端服务基地址：`http://localhost:8081`

### 4.7 启动桌面端前端（开发模式）

管理员端前端在 `frontend/desktop-vue` 目录执行：

```bash
npm install
npm run dev
```

默认开发地址为：`http://localhost:5173`

### 4.7 启动移动端 Web 前端（开发模式）

Vue 版移动端前端在 `frontend/mobile-vue` 目录执行：

```bash
npm install
npm run dev
```

默认开发地址为：`http://localhost:5174`

说明：

1. `desktop-vue` 与 `mobile-vue` 均为独立前端工程，开发模式下需要单独启动。
2. 两个前端工程通过 Vite 代理访问后端 `/api`，因此后端服务需要先启动。
3. 生产部署时，使用 Nginx 统一代理，无需单独启动前端开发服务器。

### 4.9 使用 Android 原生端

Android 原生端代码位于 `Android/` 目录，当前仓库内已包含完整 App 模块代码，例如：

1. `Android/app/build.gradle.kts`
2. `Android/app/src/main/AndroidManifest.xml`
3. `Android/app/src/main/java/com/example/myapplication/MainActivity.kt`
4. `Android/app/src/main/java/com/example/myapplication/QuickAnalyzeOverlayService.kt`
5. `Android/app/src/main/java/com/example/myapplication/AccessibilityAutoAnalyzeService.kt`

Android 端有两种使用方式：

1. 自行构建：
   - 使用 Android Studio 打开 `Android/` 目录
   - 根据需要调整 `Android/app/src/main/assets/config/app_config.json`
   - 连接真机或启动模拟器后进行编译、安装和运行
2. 直接安装：
   - 直接安装我们提供的 APK 进行演示
   - 这种方式更适合比赛演示或快速复核，不需要现场重新编译

说明：

1. Android 原生端主要承接悬浮窗快捷识别、后台提醒、无障碍自动守护等主动防护能力。
2. 基础联调可只启动后端与 Web 前端；完整展示“事中守护”时使用 Android 原生端。

## 五、账号准备

建议准备三类账号：

1. 普通用户：用于多模态分析、风险历史、聊天辅助、主动提醒演示。
2. 守护人用户：用于家庭联动通知演示。
3. 管理员用户：用于案例审核、知识库运营和后台统计演示。

家庭联防演示建议至少准备 2 个普通用户并提前建立守护关系。

## 六、核心使用路径

### 6.1 快速识别

调用 `POST /api/scam/image/quick-analyze`，输入单张截图，系统同步返回 `risk_level` 与 `reason`。

### 6.2 多模态深度分析

调用 `POST /api/scam/multimodal/analyze`，输入文本并可附带图片、音频或视频内容。系统异步完成：

1. 子智能体多模态取证
2. 主智能体工具编排
3. 当前案件评分
4. 用户历史与画像读取
5. 动态风险分级
6. 最终结构化报告
7. 历史归档与高风险联动

### 6.3 风险历史与总览

普通用户可通过：

1. `GET /api/scam/multimodal/history`
2. `GET /api/scam/multimodal/history/overview`

查看历史案件、风险等级分布与风险趋势。

### 6.4 家庭联防

高风险案件产生后，守护人可通过家庭通知 WebSocket 接收即时提醒。

### 6.5 管理端治理

管理员可通过以下接口完成案例治理：

1. `GET /api/scam/review/cases`
2. `GET /api/scam/review/cases/:recordId`
3. `POST /api/scam/review/cases/:recordId/approve`

并可继续查看案件库总览、图谱与区域风险态势。

## 七、启动后核对项

服务启动完成后，建议按以下顺序核对关键能力是否可用：

1. 后端接口可访问，`/api` 请求返回正常。
2. `frontend/desktop-vue` 可正常登录并进入管理员端页面。
3. `frontend/mobile-vue` 可正常登录并进入移动端页面。
4. 单图快速识别接口可返回 `risk_level` 与 `reason`。
5. 多模态深度分析任务可正常创建、完成并生成结构化结果。
6. 风险历史与总览接口可正常返回历史记录与趋势数据。
7. 家庭联防通知链路可正常触发。
8. Android 原生端的悬浮窗、后台提醒或无障碍自动守护能力可按预期运行。

## 八、补充说明

本项目的赛题核心亮点不在附加聊天能力，而在深度分析主链路中对“当前案件上下文 + 用户历史记忆 + 用户画像”的联合建模。评委如需核对赛题中“长短期记忆机制”的实现，请优先查看：

1. 主智能体工具闭环
2. 动态风险分级逻辑
3. 测评集与评估报告

## 九、结论

当前项目已具备完整后端服务、多模态深度分析、主动预警、家庭联防和知识治理能力。本文档的目标是确保比赛提交包中的可运行部分能够被快速部署和稳定演示。
